# TicketEscape

`TicketEscape` は、`https://escape.id/*` の購入フォームを指定時刻に自動操作する Chrome 拡張です。  
API 連携ではなく、実際のページ上のボタン・チェックボックスを `click()` して動作します。

## 0. このREADMEの対象

このREADMEは、次のような方を想定しています。

- Chrome拡張を手動で入れたことがない
- GitHubからダウンロードしたことがあまりない
- 「とにかく失敗しない手順」を上から順番に知りたい

「まず入れて、次に設定して、最後に当日運用する」までを1本で説明します。

## 1. 先に知っておくこと（重要）

- この拡張は Chrome Web Store 版ではありません。
- GitHub からダウンロードして、自分で Chrome に読み込みます。
- 対象は `https://escape.id/*` のページのみです。
- 実行前に `escape.id` へログインしておく必要があります。
- サイト側のHTML構造変更で動かなくなることがあります。

## 2. できること

- 対象URLの保存
- JST指定時刻での自動実行
- 券種ごとの数量調整
- フォーム内チェックボックスの自動ON
- カート投入ボタン押下
- 実行並列数（同時タブ数）の設定（1〜5）

## 3. 動作環境

- Google Chrome（最新版推奨）
- macOS / Windows
- このプロジェクトフォルダ（`manifest.json` を含む）

## 4. インストール手順（初心者向け）

### 4.1 ファイルをダウンロードする

### 方法A: ブラウザでZIPをダウンロード（おすすめ）

1. GitHubのリポジトリページを開く
2. 緑色の `Code` ボタンをクリック
3. `Download ZIP` をクリック
4. ダウンロードしたZIPを解凍する
5. 解凍後のフォルダを開き、`manifest.json` があることを確認

失敗しやすい点:

- ZIPのままでは読み込めません（必ず解凍）
- 1つ上の階層フォルダを選ぶと読み込み失敗します
- 必ず `manifest.json` が見えるフォルダを選びます

### 方法B: Gitコマンドで取得（慣れている方向け）

```bash
git clone https://github.com/FukaseDaichi/TicketEscape.git
cd TicketEscape
```

### 4.2 Chromeに読み込む

1. Chromeで `chrome://extensions/` を開く
2. 右上の `デベロッパー モード` を ON
3. `パッケージ化されていない拡張機能を読み込む` をクリック
4. `manifest.json` があるフォルダを選択
5. 一覧に `TicketEscape` が出れば成功

### 4.3 オプション画面を開く

1. `chrome://extensions/` で `TicketEscape` を探す
2. `詳細` を開く
3. `拡張機能のオプション` をクリック

または、拡張アイコンのポップアップから `拡張機能の設定を開く` を押しても開けます。

## 5. 最初の設定（本番前に必ず1回）

オプション画面には「基本設定」と「詳細設定（通常は変更不要）」があります。

### 5.1 基本設定

1. `対象URL` を入力
2. `実行時刻 (JST)` を入力
3. `確認` ボタンを押す（券種を取得）
4. `チケット数量設定` で各券種の数量を設定
5. `実行待機` を押す

保存が成功すると:

- ステータスが更新されます
- カウントダウンが始まります
- 設定した時刻に対象URLへ遷移します

### 5.2 詳細設定（必要な場合のみ変更）

| 項目                                   | 初期値 | 変更範囲/意味                                                 |
| -------------------------------------- | ------ | ------------------------------------------------------------- |
| `クリック間隔(ms)`                     | `30`   | ボタン連打の間隔。通常は変更不要。                            |
| `並列実行数`                           | `1`    | `1`〜`5`。同時に実行するタブ数。`5` なら最大5タブへ同時配布。 |
| `フォーム内チェックボックスをすべてON` | ON     | ON時、フォーム内チェックボックスを自動チェック。              |

## 6. 当日の使い方（失敗を減らす運用）

### 6.1 実行前チェック（開始10〜15分前）

1. `escape.id` にログイン済みか確認
2. 対象公演ページが表示できるか確認
3. オプション画面でURLと時刻を再確認
4. `確認` を押して券種が取得できるか確認
5. 数量を再確認して `実行待機`
6. カウントダウンが動いていることを確認

### 6.2 実行直前の注意

- PCをスリープさせない
- ネットワークを切らない
- 不要な重い処理をなるべく避ける
- オプション画面タブは開いたままにする

### 6.3 実行時の挙動

- 0秒になるとオプション画面タブが対象URLへ遷移
- 同時にバックグラウンド処理が実行開始
- 指定した並列数だけタブを使って処理を配布
- 各タブでフォーム操作が走る

## 7. ポップアップで確認できること

拡張アイコンを押すと次を確認できます。

- 保存中のURL
- 実行時刻（JST）
- 設定画面を開くボタン

詳細な設定変更はポップアップではなくオプション画面で行います。

## 8. トラブルシューティング（初心者向け）

### 8.1 インストールできない

原因と対処:

- `manifest.json` のないフォルダを選んでいる
- 対処: `manifest.json` が見える階層を選び直す

- ZIPを解凍していない
- 対処: ZIPを展開してから再読み込み

- `デベロッパー モード` がOFF
- 対処: `chrome://extensions/` 右上でONにする

### 8.2 `確認` が失敗する

原因と対処:

- URLが `https://escape.id/*` ではない
- 対処: 正しい購入ページURLを入れる

- ログインしていない
- 対処: 同じChromeプロファイルでログイン後に再試行

- ページがまだ読み込み途中
- 対処: 数秒待ってもう一度 `確認`

### 8.3 実行されない / 遅れた

原因と対処:

- 実行時刻が過去
- 対処: 時刻を再設定して `実行待機` をやり直す

- 保存後に設定を変えただけで再待機していない
- 対処: 変更後は必ず再度 `実行待機`

- PCがスリープした
- 対処: 本番前はスリープ無効にして運用

### 8.4 数量が想定と違う

原因と対処:

- 券種名の表記ゆれ
- 対処: `確認` を押し直し、取得直後の名称を使う

- サイト側UIの変更
- 対処: `確認` の結果が正しいか都度確認

## 9. 更新方法

### 9.1 ZIPで導入した場合

1. 新しいZIPをダウンロードして解凍
2. `chrome://extensions/` を開く
3. `TicketEscape` の `更新` を押す
4. 反映しない場合は削除して再読込

### 9.2 Gitで導入した場合

```bash
cd <YOUR_REPO>
git pull
```

その後、`chrome://extensions/` で `更新` を押します。

## 10. アンインストール

1. `chrome://extensions/` を開く
2. `TicketEscape` の `削除` を押す
